# shadow-gentcore

Multi-domain AI agent framework SDK. Agents are YAML + prompts, not code. Genesis agents auto-generate domains from your repos.

## Quick Start

```bash
# Install
python3.11 -m venv .venv && source .venv/bin/activate
pip install -e ../agent-contracts && pip install -e ../agent-tools && pip install -e ".[dev]"

# Verify
./ai workspace
.venv/bin/pytest harness/tests/ -q   # 862 tests

# Auto-generate a domain from your repos
./ai genesis build --team backend-team --dry-run

# Run a workflow
./ai run workflow ./domains/my_service/workflows/feature.yaml --dry-run
```

## Architecture

```
agent-contracts   — Shared Pydantic types (zero deps)
       ↑
shadow-gentcore   — SDK: engine, CLI, genesis agents, rules
       ↑
agent-tools       — Tool adapters + 42 packs (GitHub, AWS, Jira, etc.)
       ↑
domain-{team}     — Team domains (generated by genesis, customized by teams)
```

## Key Concepts

- **Agent = YAML manifest + system prompt** — no code per agent
- **Genesis agents** scan repos and auto-generate domains with agents, workflows, and harness
- **3-Layer Knowledge**: standards.md (always injected) + reference/*.md (on-demand) + tools (~10 generic)
- **RuleEngine**: 6-layer permission merge, hot-reloadable, non-negotiable platform safety
- **Reference vs Target**: genesis learns standards from your BEST repos, enforces on all
- **Agent Lifecycle States**: typed state machine (SPAWNING→READY→RUNNING→COMPLETED) for all agents
- **Typed Execution Events**: enum-based workflow events for observability
- **Trusted Paths**: teams with `trusted: true` skip permission prompts
- **Output Consistency**: Schema injection in prompts + `OutputParser` 4-strategy JSON extraction + `AnthropicProvider` structured output via `submit_output` tool forcing
- **Progressive Tool Disclosure**: Manifest whitelist (96.9% token reduction) + L1→L2 promotion via `ToolDisclosureRouter`
